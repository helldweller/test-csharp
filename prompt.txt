Ты — senior С#-разработчик и архитектор backend-систем, хорошо знакомый с C#, .net9 и frontend.

Помоги создать новый проект в Visual Studio Code.

Контекст проекта:
- Язык: c#
- Тип проекта: демонстрационные приложения, frontend и backend
- Уровень: production-ready

Создать два отдельных проекта:
- Фронтенд: Blazor WebAssembly
- Бэкенд: .NET 9 Web API (веб-сервис).

Бэкенд должен использовать SignalR и HTTP для асинхронной работы:
- Фронтенд будет отправлять запросы через HTTP, а ответы получать через SignalR.
- На фронтенде создать текстовое поле и кнопку "Отправить".

Когда пользователь вводит текст и нажимает кнопку, фронтенд должен отправить запрос на сервер (через HTTP).
Сервер обрабатывает запрос, добавляет текущее время к тексту и отправляет обратно через SignalR.
На фронтенде, получив ответ, необходимо вставить его в специальный div (отдельный контейнер).
Каждый новый ответ будет перезаписывать содержимое этого контейнера.

Технические детали:
- Использование Blazor для фронтенда.
- Использование .NET 9 для бэкенда.

Использование SignalR для двусторонней связи между фронтендом и бэкендом.
На сервере создание простого метода, который будет получать текст, добавлять текущее время и отправлять ответ через SignalR.

Дополнительные требования:
- Использование асинхронных вызовов для работы с сервером.
- Структура проекта должна быть чистой и разделять логику фронтенда и бэкенда.
- Код должен быть структурирован и готов к расширению.

Ты — senior С#-разработчик и архитектор backend-систем, хорошо знакомый с C#, .NET 9 и frontend.

Помоги создать новый проект в Visual Studio Code.

Контекст проекта:
- Язык: C#
- Тип проекта: демонстрационное приложение, frontend и backend
- Уровень: production-ready

Создать два отдельных проекта:
- Фронтенд: Blazor WebAssembly.
- Бэкенд: .NET 9 Web API (веб-сервис).

### Бэкенд:
- Бэкенд должен использовать SignalR и HTTP для асинхронной работы.
- На сервере создать простой API-метод, который будет получать текст, добавлять текущее время и отправлять ответ через SignalR.
- Использование асинхронных вызовов на сервере для обработки запросов.
- Необходимо обеспечить логирование на сервере для отслеживания ошибок и производительности.
- Возможность масштабирования сервера при увеличении нагрузки (например, использование пулов соединений SignalR).

### Фронтенд:
- На фронтенде создать текстовое поле и кнопку "Отправить".
- Когда пользователь вводит текст и нажимает кнопку, фронтенд должен отправить запрос на сервер (через HTTP).
- Получив ответ от сервера через SignalR, необходимо обновить специальный контейнер (div) с новым текстом, добавляя время.
- Каждый новый ответ перезаписывает содержимое контейнера.
- Использовать `StateHasChanged` для обновления UI в Blazor.
- Обработка ошибок на фронтенде: если сервер не доступен, необходимо показывать пользователю сообщение об ошибке.

### Дополнительные требования:
- Структура проекта должна быть чистой и разделять логику фронтенда и бэкенда.
- Код должен быть структурирован и готов к расширению (например, возможность добавления новых API-методов).
- Реализовать обработку ошибок и возможность повторных попыток при неудачных запросах на сервер.
- Сервер должен быть готов к масштабированию и обработке большого числа асинхронных соединений через SignalR.

### Ожидаемый результат:
1. Структура проектов: два независимых проекта — Blazor (Frontend) и .NET 9 Web API (Backend).
2. Фронтенд и бэкенд используют HTTP для запросов и SignalR для получения ответов в реальном времени.
3. Простая система взаимодействия с сервером и обновления данных на фронтенде.
4. Чистый и масштабируемый код.
